# AI噪音模式库

> 此文件记录常见的AI生成痕迹，帮助识别和避免"AI味"。
> 来源：参考khazix-writer的禁用规则，结合实际使用经验持续更新。

## 禁用词汇

### 高频踩雷词（绝对禁用）
- **说白了** ← AI特别爱用，一出现立刻暴露
- **意味着什么？** ← AI标志性句式
- **这意味着** ← 同上，换成更口语的表达
- **本质上** ← 太学术
- **换句话说** ← 太书面
- **不可否认** ← 套话
- **综上所述** ← 典型的总结套话
- **总而言之** ← 同上
- **值得注意的是** ← 书面腔
- **不难发现** ← 书面腔
- **与此同时** ← 书面腔

### 套话结构（避免使用）
- **首先...其次...最后** ← 过度结构化
- **让我们来看看...** ← 教科书式开头
- **接下来让我们...** ← 同上
- **在当今...的时代** ← 空话开头
- **随着...的发展** ← 空话开头
- **在深入探讨之前** ← 拖沓

## 禁用标点

### 绝对禁用
- **冒号"："** → 用逗号代替
- **破折号"——"** → 用逗号或句号替代
- **双引号""或""** → 用「」替代，或直接不加引号（对话除外）

### 使用说明
这些标点在书面文章中常见，但在口播逐字稿中会打断节奏。用逗号和句号可以让文字更接近说话的自然停顿。

## 过度结构化表现

### 连续bullet point
- 连续使用3个以上的bullet point罗列观点 → 改成散文叙述
- 大段加粗（超过2行） → 过度强调，反而分散重点

### 小标题滥用
- 不是每个段落都需要小标题
- 口播逐字稿应该用自然转场，而非标题切割

## 假设性例子

### 错误示例
- "比如有一次..." ← 编造场景是大忌
- "假设你遇到..." ← 假设性场景缺少真实感
- "想象一下..." ← 同上

### 正确做法
- 用正在发生的真实细节："就像我今天正在搞的xxx"
- 如果没有真实细节，不如写："我自己还没试过，但想想就觉得xxx"
- 承认不确定性比编造更真诚

## 空泛表达

### 工具名空泛
- ❌ "AI工具"、"某个模型"、"相关技术"
- ✅ 具体名字：Claude Code、Codex、Seedance 2.0、Deepresearch

### 情绪空泛与夸张
- ❌ "我感到非常震撼"、"真的很重要"、"太关键了"
- ❌ "我当时就懵住了"、"我当时就愣住了" ← AI常用的体感表达，也很假
- ❌ 夸张化表达：夸张的用词、夸张的数据、夸张的情感
- ✅ 真诚且自然的表达：说清楚为什么，不说"很重要"，说"不做这个会导致XX问题"
- ✅ 克制的情绪表达：用具体行为代替情绪词，如"我停下来想了想"而非"我震惊了"

### 观点空泛
- ❌ "这个方法很有效"
- ✅ "我用这个方法3天写了5篇文章，比之前快了一倍"

## 教科书式开头

### 禁止的开头模式
- "在当今AI快速发展的时代..."
- "随着技术的不断进步..."
- "近年来，XX领域发生了巨大变化..."
- "下面我来介绍一下..."

### 正确的开头方式
- 从一个具体的、当下的事件切入
- 第一句话让读者产生"然后呢？"的冲动
- 参考：my-voice.md 中的"开头直接"原则

## 识别AI味的关键信号

### 温度感缺失与夸张化
- 情绪表达是知识性描述（"我感到非常震撼"）或AI常用体感词（"我当时就懵住了"）
- 夸张化表达：过度的情绪词、夸张的数据、夸张的用词
- 人物描写让人感觉不到"体温"
- 缺少真诚且自然的表达，过度追求"有感染力"反而显得刻意

### 姿态问题
- 不自觉地滑入"导师在教学生"或"品牌在做营销"的姿态
- 而非"一个有见识的普通人在认真聊一件打动他的事"

### 节奏呆板
- 连续3句以上句式长度相近
- 缺少一句话独立成段的"断裂"效果
- 偏离主线后没有"扣主线句"拉回来

## 使用建议

在创作初稿和迭代优化时：
1. 创作完成后，全文搜索禁用词汇，逐个替换
2. 检查标点使用，确保没有冒号、破折号、双引号
3. 检查是否有假设性例子，改用真实细节或承认不确定
4. 检查工具名、情绪表达、观点是否具体
5. 通读全文，感受是否有"AI在输出信息"的感觉
