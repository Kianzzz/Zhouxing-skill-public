# 对标账号分析与选题监控

## 前置依赖

**必须**：wechat-extractor skill 的模式 2（账号搜索）必须可用。

检查清单：
1. wechat-extractor 已安装且可用
2. 已完成微信公众号后台登录（`~/.wechat-extractor/config.json` 存在且有效）— 用于搜索账号和获取文章列表

如任一条件不满足，**立即停止并告知用户缺少什么**，不要继续执行。

## 配置管理

配置文件：`~/.claude/benchmark-monitor-config.json`

```json
{
  "storage_path": "/用户指定的Obsidian路径/",
  "configured_at": "2026-02-10"
}
```

### 首次使用

1. 检查配置文件是否存在
2. 不存在 → 用 AskUserQuestion 询问用户存储路径
3. 创建配置文件并保存

### 后续使用

1. 读取配置文件获取 storage_path
2. 检查路径是否存在，不存在则创建

### 对标账号清单管理

清单文件存放在 `{storage_path}/` 下，文件名格式：`对标清单-{清单名}.md`

- 目录下只有 1 个清单文件 → 默认使用，不询问
- 目录下有多个清单文件 → 用 AskUserQuestion 让用户选择
- 目录下没有清单文件 → 提示用户先执行功能一创建清单

## 功能一：对标账号分析

### 触发

用户提供一个或多个公众号名称，或要求管理对标账号。

### 工作流程

#### 步骤 0：判断操作模式

检查 `{storage_path}/` 下是否存在 `对标清单-*.md` 文件：

- **没有现有清单** → 直接进入「新建清单」模式
- **有现有清单** → 用 AskUserQuestion 询问用户：
  - **新建清单**：分析一个新领域的对标账号，创建新的清单文件
  - **管理现有清单**：对已有清单进行增加/移除/更新操作

**管理现有清单子流程**：
1. 如只有 1 个清单 → 默认选中；多个 → 让用户选择
2. 用 AskUserQuestion 询问操作类型：
   - **增加账号**：用户提供新的公众号名称 → 执行步骤 1-4 分析 → 将新账号插入清单 → 重新计算所有账号权重并排序
   - **移除账号**：展示清单中所有账号 → 用户选择移除哪些 → 从清单中删除 → 重新计算权重
   - **更新数据**：对清单中所有账号重新拉取最新 100 篇数据 → 重新分析 → 重新计算权重 → 覆盖更新清单（适合定期刷新，保持数据时效性）

#### 步骤 1：逐个搜索账号

对每个公众号名称，使用 wechat-extractor 的模式 2：
1. 搜索公众号 → 获取 nickname、intro（简介）
2. 获取该账号的文章列表（近 100 篇），包含标题、作者、发布日期

具体操作：使用 wechat-extractor 的 `scripts/wechat_api.py` 搜索账号、获取文章列表。

#### 步骤 2：全量精读分析

对 100 篇文章进行深度阅读分析（通过标题 + 全文内容），提取：

- **内容领域与方向**：该账号主要写什么（2-3 句话描述）
- **内容关键词**：5-8 个高频主题关键词
- **写作风格特征**：叙事型/观点型/方法论型/情感型等
- **目标读者画像**：从内容推断该账号面向什么人群
- **选题规律**：高频选题方向、常用标题模式、内容结构特征

**精读方式**：使用 wechat-extractor 的 HTTP 提取方法逐篇获取全文内容进行分析。每篇之间间隔 3 秒避免限流。

#### 步骤 3：计算对标权重

对标权重基于内容分析综合评估账号的参考价值：

- **发文频率**（25%）：近 100 篇文章的时间跨度 → 算出平均发文频率（篇/周），频率越高说明账号越活跃
- **内容深度**（30%）：基于精读判断 — 文章是否有独特观点、深入分析、系统化思考，还是浅层搬运/鸡汤堆砌
- **领域聚焦度**（25%）：100 篇文章中有多大比例集中在核心领域，越聚焦说明账号定位越清晰，参考价值越高
- **选题创新性**（20%）：是否经常提出独特视角或冷门切入点，还是跟随热点人云亦云

每个维度 0-100 分，加权后归一化。所有账号的权重分是相对排名，最高的标准化为 100。

**增加/更新模式注意**：权重必须在所有账号（新旧合并）中重新计算归一化，而非仅对新增账号单独打分。

#### 步骤 4：输出对标清单表格

按对标权重从高到低排序，生成 Markdown 表格并保存。

- 新建模式 → 创建新文件
- 管理模式 → 覆盖更新原文件，更新"生成日期"和"最后更新"

### 输出格式

文件名：`对标清单-{用户指定的清单名}.md`

## 功能二：选题监控

### 触发

用户要求查看对标账号最近的选题动向，或要求选题推荐。

### 工作流程

#### 步骤 1：加载对标账号清单

按「配置管理 > 对标账号清单管理」的规则加载清单。

#### 步骤 2：采集近 7 天数据

对清单中的每个对标账号：
1. 使用 wechat-extractor 获取文章列表
2. 筛选近 7 天发布的文章（按当前日期计算）
3. 获取每篇文章的标题、作者、发布日期

#### 步骤 3：选题聚类

将所有对标账号 7 天内的文章标题进行语义聚类：
- 提取核心话题，去掉修辞包装
- 不同标题但讨论同一话题的归为一组
- 排除纯品牌营销、活动推广类内容

#### 步骤 4：5 维评估

对每个聚类后的选题，执行 5 维评估。详见 [evaluation-framework.md](references/evaluation-framework.md)：

1. **覆盖广度**：多少个不同账号写了同一选题，覆盖越广说明需求越确定
2. **角度饱和度**：精读各账号同选题文章，判断切入角度是趋同还是多元。角度越单一 = 差异化机会越大
3. **痛点/情绪归因**：精读代表文章全文，结合当前时间背景分析核心痛点触发点和情绪驱动力
4. **时效性**：标注 🔥热点 / 🌲常青 / 🔥🌲混合，附建议发布窗口
5. **差异化空间**：综合以上分析，指出现有文章未覆盖的角度、遗漏的论点、可以深挖的方向

#### 步骤 5：生成推荐度排名

加权计算推荐度（覆盖广度 25% + 角度饱和度 25% + 痛点归因 20% + 时效性 15% + 差异化空间 15%），输出 7-9 个选题。

### 输出格式

文件名：`选题监控-{YYYY-MM-DD}.md`

```markdown

生成日期：{YYYY-MM-DD}
监控周期：{7天前日期} ~ {今天日期}
基于对标清单：{清单文件名}

## 选题推荐（按推荐度排序）

### 1. {选题名称} ⭐⭐⭐⭐⭐

| 维度 | 评估 |
```

## 重要注意事项

1. **精读是核心**：本 skill 的价值来自对文章内容的深度阅读和分析，不依赖阅读量等外部指标。每篇文章都要认真读，不能只看标题敷衍
2. **API 调用节奏**：多个账号连续搜索时，每个账号之间间隔 3-5 秒，避免触发限流。文章内容获取也需 3 秒间隔
3. **缓存利用**：wechat-extractor 有 24 小时缓存。如果同一天内多次使用，会自动命中缓存
4. **iCloud 写入**：保存文件时先写 `/tmp` 再 `cp` 到目标路径，避免 iCloud 同步锁定问题
5. **内容获取失败时**：如个别文章无法获取全文，跳过该篇继续，在最终报告中标注跳过数量。不要因为个别失败阻塞整个流程
