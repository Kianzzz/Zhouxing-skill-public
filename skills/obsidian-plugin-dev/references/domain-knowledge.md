## 插件基本组成

- `manifest.json`：插件身份证（名称、版本、描述、作者）
- `main.js`：核心代码文件
- `styles.css`（可选）：自定义样式

## 插件生命周期

- `onload()`：插件启用时初始化所有功能
- `onunload()`：插件禁用时清理资源

## 核心 UI 组件

**命令 Command**：通过命令面板（Ctrl/Cmd + P）调用的功能。

**视图 View**：展示内容的窗口/面板，可在侧边栏、新标签页或悬浮窗显示。

**丝带图标 Ribbon Icon**：左侧边栏的快捷按钮。

**设置页 Settings Tab**：让用户配置插件选项（开关、文本框、下拉框、颜色选择器、滑块等）。

**右键菜单 Context Menu**：编辑器或文件列表右键时的菜单项。

## 常见功能实现

**文件操作**：读取/修改/创建笔记、获取文件列表。

**编辑器操作**：获取选中文本、在光标位置插入内容、替换文本。

**数据存储**：保存和读取插件设置（自动持久化）。

## 界面布局选项

| 类型 | 适用场景 |
|------|----------|
| 左侧边栏 | 导航和快捷操作 |
| 右侧边栏 | 辅助信息或工具面板 |
| 新标签页 | 需要大面积展示内容 |
| 弹窗 Modal | 临时对话框、快速输入或确认 |

## 用户体验原则

- 功能易发现：提供多种入口（命令、丝带图标、右键菜单）
- 操作流畅：减少点击次数，提供键盘快捷键
- 反馈及时：操作后给予明确提示（成功/失败消息）
- 设置合理：提供必要配置项，避免过度复杂

## 性能与错误处理

- 避免频繁读写文件
- 大量数据处理时显示加载提示
- 及时清理不再使用的资源
- 捕获异常，给用户友好的错误提示
- 避免因错误导致 Obsidian 卡死

## 常见陷阱与规避

| 陷阱 | 规避方法 |
|------|----------|
| 一次性实现所有功能 | 先实现核心功能（MVP），后续迭代 |
| 不考虑使用场景决定界面 | 高频操作用丝带图标，大量信息用侧边栏/标签页 |
| 设置项过多 | 只暴露用户真正需要的，其他用合理默认值 |
| 缺少用户引导 | 首次启用显示欢迎提示，设置页提供清晰说明 |
