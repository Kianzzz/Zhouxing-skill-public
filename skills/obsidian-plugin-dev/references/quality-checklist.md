# 质量检查清单

交付插件前，逐项检查以下内容。

## 功能完整性

- [ ] 规划中的所有核心功能都已实现
- [ ] 重要功能都已实现（除非用户同意延后）
- [ ] 所有交互入口都已配置（命令、图标、菜单等）
- [ ] 设置页包含所有必要的配置项

## 稳定性

- [ ] 插件启用和禁用正常，无报错
- [ ] 核心功能在正常使用场景下无 bug
- [ ] 异常输入不会导致崩溃
- [ ] 不会影响 Obsidian 的正常运行

## 用户体验

- [ ] 界面布局清晰美观
- [ ] 操作流程流畅自然
- [ ] 功能易于发现和使用
- [ ] 操作后有明确反馈（成功/失败提示）
- [ ] 设置项有清晰的说明文字

## 代码质量

- [ ] 代码符合 Obsidian 插件规范
- [ ] 文件结构完整（manifest.json、main.js 等）
- [ ] 无明显的性能问题
- [ ] 资源正确清理（onunload 中释放资源）

## 文档完整性

- [ ] manifest.json 信息完整准确
- [ ] 安装步骤清晰易懂
- [ ] 使用指引详细明确
- [ ] 设置说明通俗易懂

## 质量评分标准

| 评分 | 说明 |
|------|------|
| 9-10 | 生产就绪，可直接交付 |
| 7-8 | 基本可用，建议优化 |
| 5-6 | 需要修复问题后再交付 |
| <5 | 返回重新开发 |

## 自检流程

1. 模拟用户操作，测试所有功能
2. 检查边缘情况（空输入、大量数据等）
3. 验证设置保存和读取
4. 确认界面在不同尺寸下正常显示
5. 检查是否有未处理的错误
