# 内容分析与配图判断规则

## 核心原则

配图数量和位置不基于文章字数，而是基于：
1. **内容结构**：文章的小节和论点
2. **可视化价值**：该分论点是否适合用图表现
3. **主题类型**：不同文章类型的配图策略不同

---

## 文章类型分类

### Type 1: 科普 (Popular Science)

**特征**
- 讲解概念、原理、知识点
- 有多个知识点要说明
- 适合用图表现复杂概念

**配图策略**
```
✅ 适合配图
  - 概念图解
  - 原理示意
  - 对比图表
  - 流程图
  - 视觉层次示意

❌ 不配图
  - 理论推导过程
  - 抽象定义
```

**配图价值评分**
- 讲解具体现象的小节：9-10
- 有步骤的原理说明：7-8
- 纯概念定义：3-4

**建议配图数**
- 平均每2-3个小节配1张图
- 关键概念后必配图

---

### Type 2: 教程 (Tutorial/How-to)

**特征**
- 教用户如何做某件事
- 有明确的步骤
- 每步有具体的结果或产出

**配图策略**
```
✅ 适合配图
  - 每个重要步骤后
  - 预期结果展示
  - 常见错误示例
  - 工具/材料展示

❌ 不配图
  - 背景理论
  - 材料列表（可配可不配）
```

**配图价值评分**
- 有输出结果的步骤：9-10
- 需要视觉参考的操作：8-9
- 纯文字说明的步骤：5-6

**建议配图数**
- 多张（步骤越多越需要）
- 最好每个主要步骤都有

---

### Type 3: 产品评测 (Product Review)

**特征**
- 对产品进行评价和对比
- 有多个评测维度（外观、功能、性能等）
- 包含实际使用展示

**配图策略**
```
✅ 适合配图
  - 产品整体展示
  - 细节特写
  - 参数对比表
  - 使用场景
  - 性能演示

❌ 不配图
  - 品牌故事背景
  - 市场分析
```

**配图价值评分**
- 产品展示/细节展示：9-10
- 功能演示：8-9
- 价格/规格对比：7-8
- 购买建议/总结：3-4

**建议配图数**
- 中等量（3-5张）
- 重点产品和对比维度配图

---

### Type 4: 旅行/体验 (Travel/Experience)

**特征**
- 分享旅行见闻或体验
- 包含多个地点/事件
- 重点在体验和情感

**配图策略**
```
✅ 适合配图
  - 景点展示
  - 旅行路线图
  - 美食/特色展示
  - 住宿环境
  - 体验场景

❌ 不配图
  - 交通攻略
  - 抽象感受
  - 成本预算细节
```

**配图价值评分**
- 景点/环境展示：9-10
- 特色美食/体验：8-9
- 路线/地点介绍：7-8
- 感受描述：4-5

**建议配图数**
- 多张（每个重要地点/体验）
- 视觉效果为重点

---

### Type 5: 故事/叙事 (Story/Narrative)

**特征**
- 讲述一个故事或事件
- 有情节发展和转折
- 包含人物、冲突、场景

**配图策略**
```
✅ 适合配图
  - 主要角色/人物
  - 关键场景
  - 故事转折点
  - 环境氛围
  - 重要细节

❌ 不配图
  - 背景铺垫过多
  - 抽象感受
  - 哲学思考
```

**配图价值评分**
- 戏剧化场景：9-10
- 关键冲突时刻：8-9
- 人物/环境介绍：7-8
- 反思/总结：2-3

**建议配图数**
- 中等量（3-5张）
- 故事转折点和高潮部分

---

### Type 6: 观点评论 (Opinion/Commentary)

**特征**
- 作者发表观点或评价
- 主要是论述和论证
- 可能包含数据支撑

**配图策略**
```
✅ 仅封面配图
  - 不配正文

⚠️ 特殊情况
  - 如有具体数据/对比：可配1-2张
  - 引用具体事件：可配该事件的配图
```

**配图价值评分**
- 纯观点论述：1-3（不配图）
- 数据对比：6-7（可配）
- 具体案例：7-8（可配）

**建议配图数**
- 仅封面
- 例外：如有明显的对比/数据可配1张

---

### Type 7: 新闻报道 (News)

**特征**
- 报道新闻事件
- 包含事实和背景
- 注重时效性

**配图策略**
```
✅ 可配图
  - 事件现场/关键画面
  - 涉及人物
  - 地点/建筑

❌ 不配图
  - 背景信息
  - 行业动态分析
```

**配图价值评分**
- 新闻亮点/关键画面：8-9
- 事件参与人物：7-8
- 背景信息/分析：3-4

**建议配图数**
- 1-2张
- 重点放在新闻亮点本身

---

### Type 8: 其他 (Other)

根据内容特性灵活判断。

## 配图类型判断：场景图 vs 信息图

### 判断流程

对每个有配图价值的分小节，先判断配图类型，再决定设计方向：

```
该段落内容是什么？
  → 案例/故事/事件/观点/情感 → 场景图
  → 干货/方法/步骤/数据/要点总结 → 信息图
  → 混合型 → 场景图优先
```

### 场景图设计要点

**主体**：人物 + 场景
**适用场景**：叙事、案例、观点表达

设计步骤：
1. 提取段落中的核心事件/场景
2. 确定画面中的人物状态（动作、姿态、情绪）
3. 确定环境/背景
4. 将以上要素填入风格提示词的可定制变量

**示例**：
- 段落内容：「朋友帮忙后，对方承诺请吃饭却再没提过」
- 场景设计：一人独坐在餐厅，看着手机上的对话框，空荡荡的对面座位
- 人物状态：独坐、看手机、若有所思
- 环境：餐厅角落、对面空座

### 信息图设计要点

**主体**：关键信息 + 知识卡片布局
**适用场景**：干货总结、方法论、步骤、数据对比

设计步骤：
1. 提取该段落的核心要点（3-5个）
2. 设计卡片布局：标题 + 要点列表/编号 + 图标
3. 人物/场景作为背景点缀（占画面 20-30%）
4. **⚠️ 必须保持风格统一**：
   - 配色与风格库一致（如果风格要求蓝色 #0015e4，信息图也用蓝色）
   - 线条质感一致（如果风格要求手绘线条，信息图也用手绘线条）
   - 整体视觉与同篇文章的场景图协调

**示例**：
- 段落内容：「三件事：承诺要兑现、不拿难处开玩笑、看场合分享好消息」
- 信息图设计：三栏卡片布局，每栏一个要点 + 配套图标
- 点缀元素：角落里的手绘人物小图
- 配色：统一蓝色线条 + 白色背景

---



对每个分小节评分（1-10）：

| 分数范围 | 描述 | 决策 | 示例 |
|---------|------|------|------|
| **9-10** | 强烈推荐 | ✅ 必配 | 具体场景、步骤结果、美食、产品展示 |
| **7-8** | 建议配 | ✅ 应配 | 知识点说明、工具展示、地点介绍 |
| **5-6** | 可配可不配 | ⚠️ 可选 | 概念说明、数据对比（文字表达也清楚） |
| **3-4** | 不太适合 | ❌ 不配 | 理论推导、背景铺垫、市场分析 |
| **1-2** | 完全不适合 | ❌ 不配 | 纯观点、抽象定义、成本列表 |

### 评分维度

评分时考虑三个维度：

1. **可视化程度**（权重 50%）
   - 该内容是否适合用图表现？
   - 图能否显著提升理解？

2. **内容重要性**（权重 30%）
   - 这是关键内容吗？
   - 是否值得强调？

3. **阅读体验**（权重 20%）
   - 读者是否期待看到图？
   - 图能否提升阅读舒适度？

## 特殊情况处理

### 情况 1: 列表或并列内容

```markdown
## 购物清单
- 番茄
- 洋葱
- 大蒜
```

**决策**：不配图（这是材料清单，文字即可）
**评分**：2-3

### 情况 2: 代码或技术细节

```
## 安装步骤
执行命令：npm install ...
```

**决策**：不配图（代码块本身就是最佳表现）
**评分**：2-3

### 情况 3: 数据对比

```markdown
## 性能对比
产品A: 99.5分
产品B: 95分
```

**决策**：可配（如果数据很重要，可生成对比图）
**评分**：6-7

### 情况 4: 故事中的细节描写

```markdown
## 第一次见面
她穿着一件蓝色连衣裙，头上戴着白色帽子...
```

**决策**：配（这是场景的视觉核心）
**评分**：8-9

## 配图数量最终判断

### 算法

```
1. 统计所有分小节
2. 对每个分小节评分
3. 过滤评分 >= 7 的小节
4. 评分 5-6 的小节：判断是否有多余配图预算

配图数 = 评分 >= 7 的小节数 + (评分 5-6 中的选中数)

特殊规则：
- 总数 > 10 → 可能过多，需要人工审查
- 总数 < 1（仅封面情况） → 只有封面，无正文配图
```

### 示例

**文章：如何选择跑鞋（教程类）**

```
H2: 跑鞋的重要性 → 评分 4 ❌
H2: 跑步方式分类 → 评分 6 ⚠️
H2: 脚型分析方法 → 评分 7 ✅
H2: 选择步骤演示 → 评分 9 ✅
H2: 品牌推荐 → 评分 5 ⚠️
H2: 价格对比 → 评分 5 ⚠️
H2: 购买建议 → 评分 3 ❌

配图数 = 2（评分9、7）+ 1（评分5的选中）= 3张
```
