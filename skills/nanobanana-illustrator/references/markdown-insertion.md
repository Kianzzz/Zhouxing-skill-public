# Markdown 插入规范

## 核心规则

1. ✅ 使用相对路径（从文章位置指向图片目录）
2. ✅ Alt text 使用分小节标题
3. ✅ 不修改其他内容，仅插入图片
4. ✅ 图片前后保持自然的段落流

---

## 相对路径计算

### 原理

Markdown 中的图片路径相对于 `.md` 文件所在目录。

### 示例

**文件结构**：
```
Obsidian2025/
├── 📚 30-积累/
│   └── 301-指令/
│       └── 3012-视觉提示词/
│           └── 我的文章.md          ← 文章位置
│
└── 📤 40-输出/
    └── 401-图片/
        └── 我的文章-配图-1.png      ← 图片位置
```

**相对路径计算**：
```
文章：.../30-积累/.../3012-视觉提示词/我的文章.md
图片：.../40-输出/401-图片/我的文章-配图-1.png

从文章指向图片：
../../40-输出/401-图片/我的文章-配图-1.png

验证：
我的文章.md 的目录是：3012-视觉提示词/
- ../../ 到达 30-积累/ 的上级（Obsidian2025/）
- 40-输出/401-图片/ 找到图片目录 ✅
```

### 路径计算器

```python
def calculate_relative_path(article_path, image_path):
    """
    计算从文章到图片的相对路径
    """
    import os
    from pathlib import Path

    article_dir = os.path.dirname(article_path)
    relative_path = os.path.relpath(image_path, article_dir)

    # 统一使用正斜杠（Markdown 标准）
    relative_path = relative_path.replace('\\', '/')

    return relative_path

# 示例
article = "/path/to/30-积累/301-指令/3012-视觉提示词/我的文章.md"
image = "/path/to/40-输出/401-图片/我的文章-配图-1.png"
relative = calculate_relative_path(article, image)
# 结果：../../40-输出/401-图片/我的文章-配图-1.png
```

## Alt 文本（描述文本）

### 重要性

- 用户看不到图片时的替代文本
- 屏幕阅读器依赖它
- SEO 有帮助

### 编写规则

| 原则 | 示例 |
|------|------|
| 使用分小节标题 | `![脚型分析](...)` |
| 简洁清晰 | ✅ `![跑鞋选择步骤](...)` |
| 避免过长 | ❌ `![这张图片展示了如何按照科学的方法来选择适合自己脚型的跑鞋](...)` |
| 避免"图片"二字 | ❌ `![图片：脚型分析](...)` |

### 示例

```markdown
# 教程类

## 步骤1：测量脚长
![脚长测量](../../40-输出/401-图片/教程-配图-1.png)

## 步骤2：识别脚型
![脚型识别](../../40-输出/401-图片/教程-配图-2.png)

---

# 科普类

## 不同脚型的特征
![脚型分类](../../40-输出/401-图片/科普-配图-1.png)

---

# 产品评测

![细节展示](../../40-输出/401-图片/评测-配图-2.png)

## 插入位置规范

### 一般规则

```markdown
## 某个分小节标题

[前半部分内容，3-5 段]

![该小节的关键词](相对路径/图片文件名)

[后半部分内容，继续]
```

### 特殊情况处理

#### 情况 1: 小节内容较短（1-2段）

**做法**：
- 在内容末尾后插入
- 留一行空行

```markdown
## 产品外观

这款鞋子采用现代设计...

![产品外观](图片路径)
```

#### 情况 2: 小节有多个段落

**做法**：
- 在逻辑中点插入
- 通常是第2段和第3段之间

```markdown
## 脚型分析

第1段：背景介绍

第2段：具体分析开始

![脚型分析](图片路径)

第3段及以后：继续分析
```

#### 情况 3: 小节包含列表

**做法**：
- 避免在列表中间插入
- 在列表前或列表后

```markdown
## 产品特点

这款产品有以下特点：

![产品展示](图片路径)

- 特点1
- 特点2
- 特点3
```

#### 情况 4: 小节包含代码块

**做法**：
- 在代码块后插入（展示结果）
- 避免在代码块前

```markdown
## 安装步骤

执行以下命令：

```bash
npm install xxx
```

![安装结果](图片路径)

你应该看到...
```

## 文件修改规范

### 修改流程

```python
def insert_images_into_markdown(article_path, images_to_insert):
    """
    images_to_insert: [
        {
            'position': 1234,  # 文件中的字符位置
            'image_path': '../../40-输出/401-图片/xxx.png',
            'alt_text': '分小节标题'
        }
    ]
    """

    with open(article_path, 'r', encoding='utf-8') as f:
        content = f.read()

    # 从后往前插入（防止位置错乱）
    for img in reversed(images_to_insert):
        markdown_img = f"\n![{img['alt_text']}]({img['image_path']})\n"
        content = content[:img['position']] + markdown_img + content[img['position']:]

    with open(article_path, 'w', encoding='utf-8') as f:
        f.write(content)
```

### 备份保护

修改前创建备份：
```
原文件：我的文章.md
备份：我的文章.backup.md
修改后确认无误再删除备份
```

## 验证检查清单

修改完成后，检查：

```
□ 所有相对路径都正确？（用浏览器打开文件验证）
□ Alt 文本有意义吗？（与该小节关联）
□ 图片前后有足够的空行吗？
□ 没有破坏列表、代码块、表格？
□ Markdown 语法正确？（可用预览工具验证）
□ 图片文件都存在？
□ 图片文件名正确？
```

## 常见问题

### Q: 图片无法显示

**可能原因**：
1. 相对路径错误 → 验证路径
2. 图片文件不存在 → 检查图片存储目录
3. 文件编码问题 → 使用 UTF-8 编码

### Q: 插入后排版混乱

**可能原因**：
1. 在列表中间插入 → 移到列表外
2. 在代码块中插入 → 移到代码块外
3. 空行不足 → 前后各加空行

### Q: 如何调整图片大小

```markdown
# 标准大小（Markdown 原生）
![alt](path)

# 自定义大小（需要 HTML）
<img src="path" alt="alt" width="400" height="300">

# Markdown 扩展（某些编辑器支持）
![alt](path){width=400px}
```

### Q: 如何添加图片标题

```markdown
# 方法1：另起一行
![脚型分析](path)

*图1：不同脚型的特征对比*

# 方法2：HTML
<figure>
  <img src="path" alt="脚型分析">
  <figcaption>图1：不同脚型的特征对比</figcaption>
</figure>
```

## 图片署名格式

不同图片来源需要不同的署名格式，在图片引用下方添加：

### Pexels 署名

```markdown
![配图描述](相对路径/图片文件名)
*Photo by [摄影师名] on Pexels*
```

**示例：**
```markdown
![窗边的咖啡](../../images/文章名-配图-1.jpeg)
*Photo by John Smith on Pexels*
```

### TMDB 电影剧照署名

```markdown
![配图描述](相对路径/图片文件名)
*Movie still from "[英文片名]" ([年份]) via TMDB*
```

**示例：**
```markdown
![孤独的城市夜景](../../images/文章名-剧照-1.jpeg)
*Movie still from "Lost in Translation" (2003) via TMDB*
```

### 署名对比

| 来源 | 署名格式 | 是否必须 |
|------|---------|---------|
| **AI 风格图** | 无需署名 | — |
| **Pexels** | `*Photo by [摄影师] on Pexels*` | 是（Pexels 使用条款要求） |
| **TMDB 剧照** | `*Movie still from "[片名]" ([年份]) via TMDB*` | 是（TMDB 使用条款要求） |
